ElasticSearch API 3-结构化查询排序

Posted at 2016-03-02 technology  elasticsearch 

查询子句

curl -XGET 'http://localhost:9200/_search?pretty' -d '{
    "query": {
        "match": {
            "tweet": "elasticsearch"
        }
    }
}'

合并多子句

curl -XGET 'http://localhost:9200/_search?pretty' -d '{
    "query": {
        "bool": {
            "must": { "match":      { "email": "business opportunity" }},
            "should": [
                 { "match":         { "starred": true }},
                 { "bool": {
                       "must":      { "folder": "inbox" }},
                       "must_not":  { "spam": true }}
                 }}
            ],
            "minimum_should_match": 1
        }
    }
}'

查询与过滤

一般来说查询语句要比过滤语句更耗时，并且查询结果也不可缓存。

原则上来说，使用查询语句做全文本搜索或其他需要进行相关性评分的时候，剩下的全部用过滤语句

最重要的查询过滤语句

term过滤

term主要用于精确匹配哪些值，比如数字，日期，布尔值或 not_analyzed的字符串(未经分析的文本数据类型)：

 { "term": { "age":    26           }}
 { "term": { "date":   "2014-09-01" }}
 { "term": { "public": true         }}
 { "term": { "tag":    "full_text"  }}

terms过滤

terms 跟 term 有点类似，但 terms 允许指定多个匹配条件。 如果某个字段指定了多个值，那么文档需要一起去做匹配：

{
    "terms": {
        "tag": [ "search", "full_text", "nosql" ]
    }
}

range 过滤

{
    "range": {
        "age": {
            "gte":  20,
            "lt":   30
        }
    }
}

gt :: 大于
gte:: 大于等于
lt :: 小于
lte:: 小于等于

exists 和 missing 过滤

bool 过滤

{
    "bool": {
        "must":     { "term": { "folder": "inbox" }},
        "must_not": { "term": { "tag":    "spam"  }},
        "should": [
                    { "term": { "starred": true   }},
                    { "term": { "unread":  true   }}
        ]
    }
}

match_all 查询

此查询常用于合并过滤条件。 比如说你需要检索所有的邮箱,所有的文档相关性都是相同的，所以得到的_score为1

match 查询

match查询是一个标准查询，不管你需要全文本查询还是精确查询基本上都要用到它。
如果你使用 match 查询一个全文本字段，它会在真正查询之前用分析器先分析match一下查询字符：

{
    "match": {
        "tweet": "About Search"
    }
}

如果用match下指定了一个确切值，在遇到数字，日期，布尔值或者not_analyzed 的字符串时，它将为你搜索你给定的值：

{ "match": { "age":    26           }}
{ "match": { "date":   "2014-09-01" }}
{ "match": { "public": true         }}
{ "match": { "tag":    "full_text"  }}

提示： 做精确匹配搜索时，你最好用过滤语句，因为过滤语句可以缓存数据。

bool 查询

curl -XGET -i 'http://localhost:9200/website/_search?pretty' -d '{
    "query":{
        "bool": {
            "must":     { "match": { "title": "first" }},
            "must_not": { "match": { "tag":   "11" }},
            "should": [
                { "match": { "tag": "starred" }},
                { "range": { "date": { "gte": "2014/01/01" }}}
            ]
        }
    }
}'

should 查询指定文档，有则可以为文档相关性加分

带过滤的查询语句

curl -XGET -i 'http://localhost:9200/website/_search?pretty' -d '{
    "query": {
        "filtered": {
            "query":  { "match": { "title": "first" }},
            "filter": { "term": { "tags": "testing" }}
        }
    }
}'

验证查询语句

curl -XGET -i 'http://localhost:9200/_validate/query?explain&pretty' -d '{
   "query": {
      "match" : {
         "tweet" : "really powerful"
      }
   }
}'

排序

curl -XGET -i 'http://localhost:9200/website/blog/_search' -d '{
    "query":{
        "filtered":{
            "filter":{
                "query":{
                    "match":{
                        "title":"first"
                    }
                }
            }
        }
    },
    "sort":[
        {"date":{"order":"desc"}}
        {"views":{"order":"asc"}}
    ]
}'
curl -XGET -i 'http://localhost:9200/_search?sort=date:desc&sort=views&title=first'

为多值字段排序

"sort": {
    "dates": {
        "order": "asc",
        "mode":  "min"
    }
}

min, max, avg 或 sum

相关性分析

curl -XGET 'http://localhost:9200/_search?explain&format=yaml' -d '{
    "query":{
        "match":{"title":"second"}
    }
}'

curl -XGET 'http://localhost:9200/website/blog/123/_explain?format=yaml' -d '{
    "query":{
        "match":{"title":"second"}
    }
}'
