ElasticSearch API 4-索引管理

Posted at 2016-03-15 technology  elasticsearch 

索引管理

创建索引

创建索引

PUT /my_index
{
    "settings":{
        "number_of_shards":1,#主分片个数，默认5
        "number_of_replicas":0#每个主分片复制个数，默认1
    },
    "mappings":{
        "type_one":{},
        "type_two":{}
    }
}

可以通过在conifig/elasticsearch.yml中添加action.auto_create_index:false来防止自动创建索引。

删除索引

DELETE /my_index
DELETE /index_one,index_two
DELETE /index_*
DELETE /_all

设置

PUT /my_index
{
    "settings":{
        "number_of_shards":1,#主分片个数，默认5
        "number_of_replicas":0#每个主分片复制个数，默认1
    }
}

可以动态修改分片复制个数

PUT /my_temp_index/_settings
{
    "number_of_replicas":2
}

配置分析器-自定义分析器

分析器有三个顺序执行的组件结合而成（字符过滤器，分词器，标记过滤器）

  1. 字符过滤器

    可以使用html_strip 字符过滤器，来删除所有的html标签

  2. 分词器

    standard分词器 将字符串分隔成单独的字词，删除大部分标点符号
    keyword分词器 输出和它接收到的相同的字符串，不做任何处理
    whitespace分词器 只通过空格来分隔文本
    pattern分词器 可以童工正则表达式来分隔文本

  3. 标记过滤器

    lowercase\stop标记过滤器 

PUT /my_index
{
    "settings": {
        "analysis": {
            "char_filter": {
                "&_to_and": {
                    "type":       "mapping",
                    "mappings": [ "&=> and "]
            }},
            "filter": {
                "my_stopwords": {
                    "type":       "stop",
                    "stopwords": [ "the", "a" ]
            }},
            "analyzer": {
                "my_analyzer": {
                    "type":         "custom",
                    "char_filter":  [ "html_strip", "&_to_and" ],
                    "tokenizer":    "standard",
                    "filter":       [ "lowercase", "my_stopwords" ]
            }}
        }
    }
}

测试

创建索引后，用analyze API来测试新的分析器:

GET /my_index/_analyze?analyzer=my_analyzer
The quick & brown fox

使用

PUT /my_index/_mapping/my_type
{
    "properties": {
        "title": {
            "type":      "string",
            "analyzer":  "my_analyzer"
        }
    }
}

元数据

元数据中的source字段

PUT /my_index
{
    "mappings": {
        "my_type": {
            "_source": {
                "enabled":  false
            }
        }
    }
}
限定 _source 字段来请求指定字段
GET /_search
{
    "query":   { "match_all": {}},
    "_source": [ "title", "created" ]
}

元数据中的all字段

query_string 在没有指定字段时默认用 _all 字段查询

GET /_search
{
    "match": {
        "_all": "john smith marketing"
    }
}
PUT /my_index/my_type/_mapping
{
    "my_type": {
        "include_in_all": false,
        "properties": {
            "title": {
                "type":           "string",
                "include_in_all": true
            },
            ...
        }
    }
}

_all 字段仅仅是一个经过分析的 string 字段。它使用默认的分析器来分析它的值，而不管这值本来所在的字段指定的分析器。

PUT /my_index/my_type/_mapping
{
    "my_type": {
        "_all": { "analyzer": "whitespace" }
    }
}

元数据中的id字段

PUT /my_index
{
    "mappings": {
        "my_type": {
            "_id": {
                "path": "doc_id" <1>
            },
            "properties": {
                "doc_id": {
                    "type":   "string",
                    "index":  "not_analyzed"
                }
            }
        }
    }
}

警告：虽然这样很方便，但是注意它对 bulk 请求（见【bulk 格式】）有个轻微的性能影响。处理请求的节点将不能仅靠解析元数据行来决定将请求分配给哪一个分片，而需要解析整个文档主体。

别名

PUT /my_index_v1
PUT /my_index_v1/_alias/my_index

GET /*/_alias/my_index
GET /my_index_v1/_alias/*

POST /_aliases
{
    "actions": [
        { "remove": { "index": "my_index_v1", "alias": "my_index" }},
        { "add":    { "index": "my_index_v2", "alias": "my_index" }}
    ]
}

在应用中使用别名而不是索引。然后你就可以在任何时候重建索引。别名的开销很小，应当广泛使用。
